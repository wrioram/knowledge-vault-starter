# GL-001 — Canonical Home and Naming Conventions

**Type:** Static reference rule

---

## Canonical home rule

Each durable fact has one canonical home. Other files may contain summaries, indexes, logs, or wikilinks back to that source.

## Naming conventions

| Content type | Path pattern |
|---|---|
| Contact | `contacts/Full Name.md` |
| Source | `sources/Full Name.md` |
| Wiki summary | `wiki/summaries/<source-title>.md` |
| Wiki concept | `wiki/concepts/<concept-name>.md` |
| Wiki topic | `wiki/topics/<topic-name>.md` |
| Journal entry | `journal/YYYY/MM/YYYY-MM-DD HH-MM.md` |
| Meeting note | `meetings/YYYY-MM-DD - Meeting Name.md` |
| Project context | `projects/<project-name>/PROJECT_CONTEXT.md` |
| SOP | `ops/SOPs/SOP-NNN-<slug>.md` |
| Guideline | `ops/Guidelines/GL-NNN-<slug>.md` |

Keep names stable. Renames create link churn.
